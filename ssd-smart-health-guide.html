SSD HEALTH CHECK

“SMART를 확인하세요”를 실제로 하는 방법

SSD 상태는 Windows 설정 화면만으로는 충분히 보기 어렵습니다. CrystalDiskInfo 또는 SSD 제조사 도구를 열어 건강 상태·온도·총 쓰기량·오류 수치를 순서대로 확인하면 됩니다.

먼저 백업이 필요한 경우

건강 상태가 주의·나쁨·경고이거나, NVMe의 Critical Warning이 0이 아니고 읽기 오류·인식 끊김이 반복되면 수치 해석보다 중요한 파일 백업을 먼저 하세요. 복제·검사처럼 쓰기 작업이 많은 작업은 백업 뒤에 진행하는 편이 안전합니다.

1. 확인 도구를 준비하세요

가장 쉬운 방법은 무료 프로그램인 CrystalDiskInfo입니다. 설치형과 압축 해제 후 실행하는 방식이 있으며, 실행하면 연결된 저장장치 정보를 자동으로 읽습니다. 삼성·WD·Crucial·Kingston 등은 각 제조사 전용 관리 도구에서도 비슷한 정보를 제공합니다.

CrystalDiskInfo 공식 다운로드

광고성 다운로드 사이트가 아니라 공식 CrystalMark 페이지 또는 SSD 제조사 지원 페이지를 사용하세요.

무료 프로그램은 이 순서로 사용하세요

1순위: CrystalDiskInfo

가장 널리 쓰이는 무료 상태 확인 도구입니다. SATA SSD·NVMe SSD·HDD를 한 화면에서 보고, 건강 상태·온도·총 쓰기량·SMART 오류 항목을 빠르게 확인할 때 사용하세요. 모델별 메뉴를 찾기 어려운 초보자에게 가장 먼저 권합니다.

2순위: SSD 제조사 무료 관리 도구

펌웨어 업데이트와 제품별 상세 진단이 필요할 때 사용합니다. Samsung Magician, WD Dashboard, Crucial Storage Executive, Kingston SSD Manager처럼 제조사가 제공하는 무료 도구가 대표적입니다. 다만 모델에 따라 지원하지 않거나 메뉴 이름이 다를 수 있으므로, 기본 상태 확인은 CrystalDiskInfo로 먼저 해도 됩니다.

보조 확인: HWiNFO Free

온도·전력·센서 값이 함께 궁금할 때 쓰는 무료 모니터링 도구입니다. SSD의 발열이 의심될 때 유용하지만, SMART 상태를 처음 확인하는 목적이라면 CrystalDiskInfo가 더 단순합니다.

출처가 불분명한 “드라이버 자동 업데이트·레지스트리 정리·SSD 최적화” 프로그램은 진단 목적에 필요하지 않습니다. 제조사 공식 도구와 위 무료 도구만으로 먼저 확인하세요.

2. 내 SSD를 정확히 선택하세요

  1. 도구를 실행한 뒤 상단의 드라이브 목록에서 확인할 SSD를 고릅니다.
  2. 모델명·용량·드라이브 문자(C:, D:)를 보고 현재 Windows가 설치된 SSD인지 구분합니다.
  3. SSD가 여러 개라면 각각 따로 확인합니다. 외장 SSD·HDD가 함께 표시될 수 있습니다.
  4. 값이 오래된 것 같으면 프로그램의 새로 고침 기능 또는 F5를 사용합니다.

3. 화면에서 먼저 볼 4가지

건강 상태

좋음·정상이면 현재 SMART가 즉시 교체가 필요하다는 신호를 내지 않는 상태입니다. 주의·나쁨·경고이면 화면을 캡처하고 중요한 파일을 백업한 뒤 제조사 보증과 교체를 검토하세요.

온도

평소 사용 중 온도가 계속 높거나, 게임·복사 작업에서 비정상적으로 치솟으면 방열판·공기 흐름·M.2 장착 상태를 확인하세요. 온도 기준은 제품마다 다르므로 제조사 도구의 경고 기준을 우선합니다.

총 쓰기량(호스트)

총 쓰기량(호스트), Total Host Writes, Host Writes 같은 항목을 찾습니다. GB로 보이면 1,000으로 나눠 TB로 환산해 SSD 수명(TBW) 계산기에 입력할 수 있습니다. 예: 10,426GB는 약 10.4TB입니다.

오류·경고 수치

수치 이름은 SSD 제조사마다 다르지만, 아래 항목이 0이 아니거나 시간이 지날수록 증가하는지 확인하면 됩니다.

4. SSD 종류별로 이렇게 해석하세요

NVMe SSD

  • Critical Warning: 0이 아니면 경고가 기록된 상태입니다. 우선 백업하고 제조사 도구에서도 상태를 확인하세요.
  • Percentage Used / 사용률: 값이 높을수록 설계 수명에 가까워진 뜻입니다. 건강 상태와 총 쓰기량을 함께 봐야 합니다.
  • Available Spare / 예비 공간: 매우 낮거나 임계값 아래라면 교체를 준비하세요.
  • Media and Data Integrity Errors, Error Information Log Entries: 증가 추세면 백업 후 제조사 진단·보증 절차를 확인하세요.

SATA SSD·HDD

  • Reallocated Sectors Count(재할당 섹터), Current Pending Sector, Uncorrectable Sector: 0이 아니거나 증가하면 데이터 보호를 우선합니다.
  • UltraDMA CRC Error Count: SATA 장치에서 이 값만 증가한다면 SSD 자체보다 데이터 케이블·포트·접촉 문제일 수 있습니다. 케이블을 교체하고 다시 비교하세요.

5. 결과에 따라 다음 행동을 정하세요

정상·좋음이고 오류 수치도 안정적일 때

SSD 자체가 즉시 고장 신호를 내는 상태는 아닙니다. 그래도 NVMe 인식 지연·멈춤이 반복되면 슬롯 접촉, 펌웨어, BIOS, 전원 관리와 이벤트 로그를 이어서 확인하세요.

주의·경고이거나 오류 수치가 증가할 때

중요 파일부터 다른 저장장치나 클라우드에 백업하세요. 사용을 최소화하고 제조사 보증 기간·진단 결과를 확인한 뒤 교체 또는 A/S를 준비합니다.

SMART는 정상인데 증상이 계속될 때

SMART는 모든 고장을 찾아내지 못합니다. 부팅 때만 인식이 늦거나 이벤트 129·153이 반복되면 NVMe 인식 지연 점검에서 슬롯·케이블·펌웨어 순서를 확인하세요.

결과를 공유할 때 개인정보를 가리세요

상태 화면에는 SSD 모델명·일련번호·볼륨 문자 등이 보일 수 있습니다. 문의나 커뮤니티에 화면을 올릴 때는 일련번호와 PC 이름, 사용자 폴더 경로를 가린 뒤 공유하세요. 건강 상태, 온도, 총 쓰기량, 오류 항목 이름과 값만 보이면 점검에 충분합니다.

SSD 수명(TBW) 계산기NVMe 인식 지연 점검백업 저장공간 계산기

SMART는 고장 가능성을 판단하는 보조 자료입니다. “정상”만으로 모든 연결·펌웨어·컨트롤러 문제를 배제할 수는 없습니다.
