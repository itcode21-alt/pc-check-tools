에러 코드 상세

코드 34 · 장치에 필요한 설정 정보를 확인할 수 없습니다 (코드 34)

윈도우가 장치를 초기화하는 데 필요한 리소스 설정 값을 자동으로 판단하지 못했을 때 나타나는 코드로, 오래된 하드웨어나 레거시 장치에서 종종 나타납니다.

이 코드를 어떻게 해석해야 하나요?

업데이트 관련 코드는 네트워크, 캐시, 구성 요소 저장소, 디스크 공간, 서비스 상태 중 어느 지점에서 작업이 중단됐는지를 보여줍니다. 실패가 반복되는 시점과 이벤트 로그의 세부 코드를 함께 확인하면 원인을 좁히기 쉽습니다.

먼저 기록할 내용: 코드 34가 발생한 시각, 직전에 실행한 작업, 최근 설치한 드라이버·업데이트, 재부팅 뒤 같은 코드가 반복되는지를 적어 두세요.

가능성 높은 원인

  • 레거시(플러그 앤 플레이를 완전히 지원하지 않는) 하드웨어인 경우
  • BIOS 설정이 장치가 요구하는 리소스와 맞지 않는 경우
  • 드라이버가 자동 설정 값을 지원하지 않아 수동 설정이 필요한 경우
  • 다른 장치와 IRQ나 메모리 주소가 겹치는 경우
  • 매우 오래된 드라이버가 최신 시스템과 호환되지 않는 경우

첫 점검 항목

  1. 장치 제조사에 해당 모델이 현재 윈도우 버전을 공식 지원하는지 확인하세요.
  2. BIOS를 기본값으로 초기화한 뒤 재시도해 보세요.
  3. 장치 속성의 리소스 탭에서 수동으로 설정을 조정할 수 있는지 확인하세요 (고급 사용자용).
  4. 다른 PCI/PCIe 슬롯에 장치를 옮겨 리소스 충돌 여부를 확인하세요.
  5. 구형 하드웨어라면 최신 윈도우 대신 호환 모드 설치나 장치 교체를 고려하세요.

데이터와 시스템을 보호하는 주의사항

시스템 폴더 권한 변경이나 레지스트리 정리 도구 사용보다 저장 공간, 네트워크, 서비스 상태를 먼저 확인하세요.

앞 단계로 해결되지 않을 때

같은 코드가 반복되면 Windows Update 기록과 DISM·SFC 결과를 저장하고 설치 미디어의 버전이 현재 Windows와 맞는지 확인하세요.

복구, 초기화, 드라이버 제거처럼 되돌리기 어려운 작업 전에는 중요한 파일을 다른 물리 저장장치나 신뢰할 수 있는 백업 위치에 보관하세요.

자주 묻는 질문

  • 장치가 인식이 안 되면 바로 새 제품을 사야 하나요? 이 문제(코드 34)는 장치 고장보다 드라이버나 포트 설정 문제인 경우가 흔합니다. 다른 포트나 다른 PC에서 인식되는지 먼저 확인해보세요.
  • USB 드라이버를 삭제하면 어떻게 되나요? 장치 관리자에서 드라이버를 제거한 뒤 재부팅하면 Windows가 드라이버를 자동으로 다시 설치하며, 이 과정에서 인식 문제가 해결되는 경우가 있습니다.
