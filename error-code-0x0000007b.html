홈진단가이드도구게임문의

에러 코드 상세

0x0000007B · INACCESSIBLE_BOOT_DEVICE

저장장치나 부팅 경로를 읽지 못할 때 주로 연결되는 코드입니다.

이 코드를 어떻게 해석해야 하나요?

저장장치 계열 오류는 Windows가 필요한 데이터를 제때 읽지 못했거나 파일 시스템과 장치 연결에 문제가 생겼다는 단서입니다. 물리 장치 상태와 논리 파일 시스템을 분리해 봐야 합니다.

먼저 기록할 내용: 0x0000007B가 발생한 시각, 직전에 실행한 작업, 최근 설치한 드라이버·업데이트, 재부팅 뒤 같은 코드가 반복되는지를 적어 두세요.

가능성 높은 원인

  • 부팅 디스크 연결 문제
  • BIOS/UEFI 설정
  • 저장장치 드라이버 문제

첫 점검 항목

  1. 디스크 연결 상태 확인 — 되돌리기 쉬운 항목부터 확인해 위험 없이 범위를 줄입니다.
  2. BIOS 부팅 순서 점검 — 첫 확인 결과와 비교해 소프트웨어와 하드웨어 가능성을 나눕니다.
  3. 최근 저장장치 변경 내역 확인 — 앞 단계에서 해결되지 않을 때 다음 원인 후보를 검증합니다.

실제 사용자 사례

위 점검 순서로도 해결되지 않았을 때, 다른 원인으로 해결된 사례입니다.

윈도우10 → 11 업그레이드 우회로 해결한 사례

새로 조립한 PC에 윈도우 11을 USB로 직접 설치했을 때 계속 이 오류가 발생했던 사례가 있습니다. 드라이버 재설치, 복구 명령어, BIOS 설정 변경까지 다양한 시도를 했지만 해결되지 않았는데, 결국 윈도우 10을 먼저 설치한 뒤 온라인으로 윈도우 11까지 업그레이드하는 방식으로 우회해서 해결했습니다.

포인트: 정확한 원인은 명확하지 않지만, USB로 최신 윈도우를 바로 설치하는 과정에서 특정 하드웨어 조합에 문제가 생길 수 있습니다.

바이오스(CMOS) 초기화로 해결한 사례

오버클럭 설정을 건드린 직후 이 오류가 발생한 사례도 있었는데, CMOS 배터리를 잠시 분리했다가 다시 끼워 바이오스 설정을 초기화하는 것으로 해결됐습니다.

포인트: 최근 오버클럭이나 BIOS 설정을 변경한 직후 이 오류가 나타났다면, 원인이 저장장치 자체보다 불안정해진 시스템 설정에 있을 수 있습니다.

데이터와 시스템을 보호하는 주의사항

SMART 경고, 읽기 실패, Disk 7·129·153 반복이 있으면 복구 명령보다 중요한 파일 백업을 먼저 진행하세요.

앞 단계로 해결되지 않을 때

백업 후 케이블·포트·M.2 슬롯을 교차하고 제조사 진단과 펌웨어를 확인한 뒤 파일 시스템 검사를 계획하세요.

복구, 초기화, 드라이버 제거처럼 되돌리기 어려운 작업 전에는 중요한 파일을 다른 물리 저장장치나 신뢰할 수 있는 백업 위치에 보관하세요.

관련 증상 가이드 확인하기

자주 묻는 질문

  • SSD·NVMe를 바로 교체해야 하나요? 이 문제(0x0000007B)만으로 저장장치 고장을 단정하기는 어렵습니다. 슬롯 재장착과 펌웨어 업데이트를 먼저 확인한 뒤에도 반복되면 다른 슬롯이나 드라이브로 교차 테스트해보는 것이 안전합니다.
  • SSD 상태를 확인하는 무료 도구가 있나요? CrystalDiskInfo 같은 무료 프로그램으로 SSD의 건강 상태와 온도를 확인할 수 있습니다.
