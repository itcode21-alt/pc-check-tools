⚠️ data.js 수정 주의사항

⚠️ data.js는 600KB+ 대용량 파일입니다. 텍스트 에디터로 직접 편집하면 위치 오류·중복 브레이스 문제가 생기기 쉽습니다. 대량 삽입 시 Python 스크립트를 권장합니다.
Python 삽입 패턴 (권장)

특정 마커 문자열을 찾아 그 위치에 내용을 삽입하는 방식입니다.

with open('data.js', encoding='utf-8') as f: content = f.read() new_entry = ''' "new-symptom-id": { badge: "분류", ... }''' # 삽입 대상 마커 (고유한 문자열 선택) marker = ' // 부품 호환성 데이터\n parts:' replacement = new_entry + '\n' + marker content = content.replace(marker, replacement, 1) # count=1 필수 with open('data.js', 'w', encoding='utf-8') as f: f.write(content) # 반드시 구문 검사 # node --check data.js
JS 문자열 내 이스케이프 주의

JS 문자열 안에 이중 따옴표(")가 포함된 내용(PowerShell 명령, 경로 등)은 반드시 이스케이프 처리해야 합니다.

// 잘못된 예 — SyntaxError 발생 text: "Register "$($_.InstallLocation)\AppXManifest.xml"" // 올바른 예 — 이중 따옴표 이스케이프 text: "Register \"$($_.InstallLocation)\\AppXManifest.xml\"" // 또는 작은따옴표 혼용 (JS에서는 동일) text: 'Register "$($_.InstallLocation)\\AppXManifest.xml"'
⚠️ Python 스크립트로 삽입할 때 Python 문자열과 JS 문자열 두 겹의 이스케이프가 필요한 경우가 있습니다. 삽입 후 node --check data.js로 반드시 확인하세요.
닫는 브레이스 중복 주의

Python replacement 문자열 끝에 이미 닫는 }가 있는데 마커 앞에도 }가 있으면 이중으로 삽입됩니다. 삽입 후 node --check가 실패하면 중복 브레이스를 먼저 확인하세요.

🔖 캐시 버스터

모든 HTML 파일은 CSS·JS를 쿼리스트링으로 버전 관리합니다. 대규모 변경 시 아래 문자열을 전체 치환하세요.

스타일style.css?v=layout-20260723c
데이터data.js?v=cache-sync-20260721
앱 로직app.js?v=cache-sync-20260721
버전 일괄 변경 (Python)
python3 -c " import glob old = 'style.css?v=layout-20260723c' new = 'style.css?v=layout-20260801a' for f in glob.glob('*.html'): with open(f) as fp: c = fp.read() if old in c: with open(f,'w') as fp: fp.write(c.replace(old, new)) print('Done') "
⚠️ CSS·JS 내용이 실질적으로 변경된 경우에만 버전 갱신. 불필요한 변경은 전체 캐시 무효화로 이어집니다.

✅ 정기 점검 체크리스트

새 페이지 추가할 때마다
  • node --check data.js — JS 구문 오류 없음 확인

  • HTML에 og:image 있는지 (소셜 공유 썸네일)
  • HTML에 canonical URL 올바른지
  • HTML에 JSON-LD 스키마 3개(TechArticle·BreadcrumbList·FAQPage) 있는지
  • 증상 페이지라면 data-symptom-detail-page="[id]" 속성이 id와 일치하는지
  • sitemap.xml에 URL 추가했는지 (</urlset> 위에 삽입)
  • node scripts/build-search-index.mjs 실행했는지

JSON-LD 스키마 3종

검색 결과 리치 스니펫을 위해 페이지마다 아래 3개 스키마를 모두 포함해야 합니다.

① TechArticle
<script type="application/ld+json"> { "@context": "https://schema.org", "@type": "TechArticle", "headline": "페이지 제목", "description": "페이지 설명", "url": "https://itsvc.co.kr/파일명.html", "mainEntityOfPage": "https://itsvc.co.kr/파일명.html", "inLanguage": "ko-KR", "dateModified": "2026-07-23", "author": { "@type": "Organization", "name": "PC 윈도우 진단 센터", "url": "https://itsvc.co.kr/about.html" }, "publisher": { "@type": "Organization", "name": "PC 윈도우 진단 센터", "url": "https://itsvc.co.kr/" } } </script>
② BreadcrumbList
<script type="application/ld+json"> { "@context": "https://schema.org", "@type": "BreadcrumbList", "itemListElement": [ { "@type": "ListItem", "position": 1, "name": "홈", "item": "https://itsvc.co.kr/" }, { "@type": "ListItem", "position": 2, "name": "가이드", "item": "https://itsvc.co.kr/guides.html" }, { "@type": "ListItem", "position": 3, "name": "페이지 이름", "item": "https://itsvc.co.kr/파일명.html" } ] } </script>
③ FAQPage
<script type="application/ld+json"> { "@context": "https://schema.org", "@type": "FAQPage", "mainEntity": [ { "@type": "Question", "name": "자주 묻는 질문 1", "acceptedAnswer": { "@type": "Answer", "text": "답변 내용" } }, { "@type": "Question", "name": "자주 묻는 질문 2", "acceptedAnswer": { "@type": "Answer", "text": "답변 내용" } } ] } </script>
