전원·재부팅

Kernel-Power 41

홈진단가이드도구게임문의

EVENT ID 41

비정상 종료의 원인을 바로 PSU로 단정하지 마세요

Kernel-Power 41은 Windows가 정상 종료 절차를 끝내지 못한 뒤 부팅되었다는 결과 기록입니다.

정전, 멈춘 PC의 전원 버튼을 길게 누른 경우, 블루스크린 뒤 자동 재부팅, 과열과 전원 불안정이 모두 같은 이벤트를 만들 수 있습니다. 따라서 이 기록 하나만으로 파워서플라이 고장이라고 판단하면 정상 부품을 교체하고 실제 원인을 놓칠 수 있습니다.

먼저 구분할 상황

  • 화면이 멈춘 뒤 직접 껐다면 이벤트 41은 강제 종료의 결과일 가능성이 큽니다. 멈춤 직전 앱과 드라이버를 확인합니다.
  • 전원이 순간적으로 완전히 꺼졌다면 PSU, 24핀·EPS·GPU 전원, 멀티탭, 과열을 먼저 봅니다.
  • 파란 화면 뒤 재부팅됐다면 같은 시각의 BugCheck 1001과 덤프 파일을 확인합니다.

안전한 점검 순서

  1. 작업 중인 파일을 백업하고 발생 시각과 당시 부하를 기록합니다.
  2. 이벤트 41의 세부 정보에서 BugcheckCode가 0인지 확인하고, 직전 5분의 WHEA-Logger와 Display 이벤트를 봅니다.
  3. HWiNFO 같은 기록에서 CPU·GPU 온도와 전력 제한 여부를 비교합니다.
  4. 전원을 끄고 24핀 ATX, CPU EPS, GPU 보조전원의 걸쇠와 PSU 모듈러 케이블 조합을 확인합니다.
  5. XMP·EXPO와 오버클럭을 기본값으로 돌린 뒤 같은 작업에서 재현되는지 확인합니다.

하면 안 되는 실수

이벤트 41만 보고 PSU를 구매하거나 레지스트리 값을 바꾸지 마세요. 부하가 없는 유휴 상태에서도 발생하는지, 특정 게임에서만 발생하는지, WHEA 18이나 Display 4101이 함께 기록되는지가 더 중요한 단서입니다.

즉시 점검이 필요한 경우

타는 냄새, 케이블 변색, 반복되는 순간 전원 차단, 저장 중 전원 꺼짐이 있으면 사용을 줄이고 백업과 전원 계통 점검을 우선하세요.

자주 묻는 질문

  • Kernel-Power 41이 뜨면 무조건 파워서플라이(PSU) 문제인가요? 아닙니다. 이 이벤트는 정상 종료 절차 없이 재부팅됐다는 결과 기록일 뿐, 원인을 특정하지 않습니다. 블루스크린 뒤 자동 재부팅, 사용자가 강제로 전원 버튼을 누른 경우도 같은 이벤트를 남깁니다. PSU 문제로 단정하기 전에 화면이 멈췄었는지, 완전히 꺼졌었는지부터 구분하세요.
  • 블루스크린 없이 그냥 꺼졌다 켜졌는데도 41이 기록되나요? 네, 화면에 아무 오류 없이 갑자기 꺼졌다가 재부팅된 경우에도 Kernel-Power 41이 기록됩니다. 이 경우 BugcheckCode가 0으로 표시되는데, 이는 시스템이 정상 종료 코드를 남길 새도 없이 전원이 끊겼다는 뜻이라 전원 계통이나 과열 가능성을 우선 확인해야 합니다.
  • 노트북에서도 이 이벤트가 발생하나요? 네, 배터리·어댑터 문제, 과열로 인한 강제 종료, 절전 모드 전환 오류로도 노트북에서 Kernel-Power 41이 기록될 수 있습니다. 데스크톱과 달리 배터리 상태와 충전 어댑터 접촉 불량도 함께 점검 대상에 포함하세요.
