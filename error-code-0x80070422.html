에러 코드 상세

0x80070422 · 필요한 서비스가 비활성화됨

Windows Update나 관련 서비스가 중지 또는 비활성화되어 작업을 진행하지 못할 때 나타날 수 있습니다.

이 코드를 어떻게 해석해야 하나요?

업데이트·설치 코드는 파일 누락, 구성 요소 저장소, 서비스, 권한, 여유 공간 중 작업이 중단된 지점을 나타냅니다. 실패 비율과 업데이트 기록의 세부 코드를 함께 봐야 합니다.

먼저 기록할 내용: 0x80070422가 발생한 시각, 직전에 실행한 작업, 최근 설치한 드라이버·업데이트, 재부팅 뒤 같은 코드가 반복되는지를 적어 두세요.

가능성 높은 원인

  • Windows Update 서비스 중지
  • 최적화 도구가 서비스를 변경
  • 정책 또는 보안 프로그램의 차단

첫 점검 항목

  1. 서비스 상태 확인 — 되돌리기 쉬운 항목부터 확인해 위험 없이 범위를 줄입니다.
  2. 시작 유형을 기본값과 비교 — 첫 확인 결과와 비교해 소프트웨어와 하드웨어 가능성을 나눕니다.
  3. 업데이트 문제 해결사와 이벤트 로그 확인 — 앞 단계에서 해결되지 않을 때 다음 원인 후보를 검증합니다.

데이터와 시스템을 보호하는 주의사항

시스템 폴더 권한 변경이나 레지스트리 정리 도구 사용보다 저장 공간, 네트워크, 서비스 상태를 먼저 확인하세요.

앞 단계로 해결되지 않을 때

같은 코드가 반복되면 Windows Update 기록과 DISM·SFC 결과를 저장하고 설치 미디어의 버전이 현재 Windows와 맞는지 확인하세요.

복구, 초기화, 드라이버 제거처럼 되돌리기 어려운 작업 전에는 중요한 파일을 다른 물리 저장장치나 신뢰할 수 있는 백업 위치에 보관하세요.

관련 증상 가이드 확인하기

같은 증상과 연결된 다른 오류 코드

  • 0x800F081F · 원본 파일을 찾을 수 없음
  • 0x80070070 · 디스크 공간 부족
  • 0xC1900101 · 업데이트 드라이버 호환성 실패
  • 0x80240034 · WU_E_UNKNOWN 알 수 없는 업데이트 오류

자주 묻는 질문

  • 업데이트를 계속 재시도해도 괜찮은가요? 이 문제(0x80070422)로 반복 실패한다면 무한정 재시도하기보다 업데이트 구성 요소 정리와 디스크 공간 확인을 먼저 하는 것이 시간을 아낄 수 있습니다.
  • 수동으로 업데이트를 받아 설치해도 되나요? Microsoft Update 카탈로그에서 해당 KB 번호를 직접 받아 설치하면 자동 업데이트 오류를 우회할 수 있는 경우가 있습니다.
