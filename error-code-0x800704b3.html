에러 코드 상세

0x800704B3 · 원격 프로시저 호출(RPC)이 실패했습니다

네트워크로 연결된 다른 컴퓨터나 서비스와 통신하는 데 필요한 RPC(원격 프로시저 호출)가 실패했을 때 나타나는 오류입니다. 프린터 공유, 원격 관리 작업 등에서 자주 나타납니다.

이 코드를 어떻게 해석해야 하나요?

업데이트 관련 코드는 네트워크, 캐시, 구성 요소 저장소, 디스크 공간, 서비스 상태 중 어느 지점에서 작업이 중단됐는지를 보여줍니다. 실패가 반복되는 시점과 이벤트 로그의 세부 코드를 함께 확인하면 원인을 좁히기 쉽습니다.

먼저 기록할 내용: 0x800704B3가 발생한 시각, 직전에 실행한 작업, 최근 설치한 드라이버·업데이트, 재부팅 뒤 같은 코드가 반복되는지를 적어 두세요.

가능성 높은 원인

  • RPC 관련 윈도우 서비스(RPC, DCOM 서버 프로세스 실행기)가 중지된 경우
  • 방화벽이 RPC 통신에 필요한 포트를 차단하는 경우
  • 대상 컴퓨터와의 네트워크 연결이 불안정한 경우
  • 네트워크 프로필이 공용으로 설정되어 있어 통신이 제한되는 경우
  • 도메인/워크그룹 설정 불일치

첫 점검 항목

  1. 서비스 관리자(services.msc)에서 Remote Procedure Call(RPC) 서비스가 실행 중인지 확인하세요.
  2. 방화벽 설정에서 원격 관리나 파일/프린터 공유 관련 규칙이 허용되어 있는지 확인하세요.
  3. 네트워크 프로필이 '개인'으로 설정되어 있는지 확인하세요 (공용 네트워크는 일부 통신이 제한됩니다).
  4. 대상 컴퓨터와의 기본 네트워크 연결(ping)이 정상인지 확인하세요.
  5. 두 컴퓨터가 같은 워크그룹이나 도메인에 속해 있는지 확인하세요.

데이터와 시스템을 보호하는 주의사항

시스템 폴더 권한 변경이나 레지스트리 정리 도구 사용보다 저장 공간, 네트워크, 서비스 상태를 먼저 확인하세요.

앞 단계로 해결되지 않을 때

같은 코드가 반복되면 Windows Update 기록과 DISM·SFC 결과를 저장하고 설치 미디어의 버전이 현재 Windows와 맞는지 확인하세요.

복구, 초기화, 드라이버 제거처럼 되돌리기 어려운 작업 전에는 중요한 파일을 다른 물리 저장장치나 신뢰할 수 있는 백업 위치에 보관하세요.

자주 묻는 질문

  • 프린터를 재설치하면 바로 해결되나요? 이 문제(0x800704B3)는 드라이버보다 서비스나 권한 문제인 경우가 많아, 재설치 전에 스풀러 서비스 상태부터 확인하는 것이 순서상 더 빠릅니다.
  • 네트워크 프린터와 USB 프린터 중 어느 쪽이 더 자주 멈추나요? 네트워크 프린터가 네트워크 상태에 따라 추가 단계에서 더 자주 멈추는 경향이 있어, 유선 연결 상태를 먼저 확인하는 것이 좋습니다.
