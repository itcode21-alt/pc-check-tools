에러 코드 상세

코드 10 · 이 장치를 시작할 수 없습니다 (코드 10)

장치 관리자에서 특정 하드웨어가 정상적으로 시작되지 못했을 때 나타나는 가장 흔한 오류입니다. 원인이 광범위해서 오디오, 네트워크, USB 장치 등 어떤 하드웨어에서든 나타날 수 있습니다.

이 코드를 어떻게 해석해야 하나요?

업데이트 관련 코드는 네트워크, 캐시, 구성 요소 저장소, 디스크 공간, 서비스 상태 중 어느 지점에서 작업이 중단됐는지를 보여줍니다. 실패가 반복되는 시점과 이벤트 로그의 세부 코드를 함께 확인하면 원인을 좁히기 쉽습니다.

먼저 기록할 내용: 코드 10가 발생한 시각, 직전에 실행한 작업, 최근 설치한 드라이버·업데이트, 재부팅 뒤 같은 코드가 반복되는지를 적어 두세요.

가능성 높은 원인

  • 드라이버 버전이 오래되었거나 현재 윈도우 버전과 호환되지 않는 경우
  • 장치 드라이버 설치 파일이 손상된 경우
  • BIOS/UEFI에서 해당 장치 관련 기능(예: 온보드 사운드, 내장 카메라)이 비활성화된 경우
  • 전원 관리 설정이 장치를 완전히 끄고 다시 켜지 못하는 경우
  • 하드웨어 자체의 물리적 결함이나 접촉 불량

첫 점검 항목

  1. 장치 관리자에서 해당 장치를 제거한 뒤 '하드웨어 변경 사항 검색'으로 재설치해 보세요.
  2. 제조사 홈페이지에서 최신 드라이버를 받아 클린 설치해 보세요.
  3. BIOS에서 관련 장치 기능이 비활성화되어 있지 않은지 확인하세요.
  4. 장치 속성의 전원 관리 탭에서 '전원 절약을 위해 이 장치를 끌 수 있음' 옵션을 해제해 보세요.
  5. 다른 포트나 슬롯에 연결해 물리적 접촉 문제인지 확인하세요.

데이터와 시스템을 보호하는 주의사항

시스템 폴더 권한 변경이나 레지스트리 정리 도구 사용보다 저장 공간, 네트워크, 서비스 상태를 먼저 확인하세요.

앞 단계로 해결되지 않을 때

같은 코드가 반복되면 Windows Update 기록과 DISM·SFC 결과를 저장하고 설치 미디어의 버전이 현재 Windows와 맞는지 확인하세요.

복구, 초기화, 드라이버 제거처럼 되돌리기 어려운 작업 전에는 중요한 파일을 다른 물리 저장장치나 신뢰할 수 있는 백업 위치에 보관하세요.

자주 묻는 질문

  • 장치가 인식이 안 되면 바로 새 제품을 사야 하나요? 이 문제(코드 10)는 장치 고장보다 드라이버나 포트 설정 문제인 경우가 흔합니다. 다른 포트나 다른 PC에서 인식되는지 먼저 확인해보세요.
  • USB 드라이버를 삭제하면 어떻게 되나요? 장치 관리자에서 드라이버를 제거한 뒤 재부팅하면 Windows가 드라이버를 자동으로 다시 설치하며, 이 과정에서 인식 문제가 해결되는 경우가 있습니다.
