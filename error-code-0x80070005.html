홈진단가이드게임문의

에러 코드 상세

0x80070005 · Access Denied

권한 부족으로 파일이나 설정에 접근하지 못할 때 보일 수 있습니다.

이 코드를 어떻게 해석해야 하나요?

업데이트·설치 코드는 파일 누락, 구성 요소 저장소, 서비스, 권한, 여유 공간 중 작업이 중단된 지점을 나타냅니다. 실패 비율과 업데이트 기록의 세부 코드를 함께 봐야 합니다.

먼저 기록할 내용: 0x80070005가 발생한 시각, 직전에 실행한 작업, 최근 설치한 드라이버·업데이트, 재부팅 뒤 같은 코드가 반복되는지를 적어 두세요.

가능성 높은 원인

  • 관리자 권한 부족
  • 보안 소프트웨어 차단
  • 정책 제한

첫 점검 항목

  1. 관리자 권한으로 다시 실행 — 되돌리기 쉬운 항목부터 확인해 위험 없이 범위를 줄입니다.
  2. 보안 프로그램 임시 확인 — 첫 확인 결과와 비교해 소프트웨어와 하드웨어 가능성을 나눕니다.
  3. 폴더 권한 점검 — 앞 단계에서 해결되지 않을 때 다음 원인 후보를 검증합니다.

실제 사용자 사례

위 점검 순서로도 해결되지 않았을 때, 다른 원인으로 해결된 사례입니다.

드라이브 압축 해제로 해결한 사례

마이크로소프트 스토어 앱 설치가 이 오류로 계속 실패했던 사례가 있습니다. 관리자 권한, 폴더 권한 부여 등 일반적인 방법으로는 해결되지 않았는데, 앱을 설치하려던 보조 드라이브에 걸려 있던 NTFS 압축(디스크 공간 절약) 기능을 해제한 뒤 정상적으로 설치됐습니다.

포인트: C 드라이브가 아닌 다른 드라이브에 앱이나 프로그램을 설치할 때 이 오류가 난다면, 그 드라이브에 압축 기능이 켜져 있지 않은지 확인해보세요.

데이터와 시스템을 보호하는 주의사항

시스템 폴더 권한 변경이나 레지스트리 정리 도구 사용보다 저장 공간, 네트워크, 서비스 상태를 먼저 확인하세요.

앞 단계로 해결되지 않을 때

같은 코드가 반복되면 Windows Update 기록과 DISM·SFC 결과를 저장하고 설치 미디어의 버전이 현재 Windows와 맞는지 확인하세요.

복구, 초기화, 드라이버 제거처럼 되돌리기 어려운 작업 전에는 중요한 파일을 다른 물리 저장장치나 신뢰할 수 있는 백업 위치에 보관하세요.

자주 묻는 질문

  • 프린터를 재설치하면 바로 해결되나요? 이 문제(0x80070005)는 드라이버보다 서비스나 권한 문제인 경우가 많아, 재설치 전에 스풀러 서비스 상태부터 확인하는 것이 순서상 더 빠릅니다.
  • 프린터 스풀러 서비스를 재시작하면 도움이 되나요? 네, services.msc에서 Print Spooler 서비스를 재시작하면 멈춰있던 인쇄 대기열이 정리되며 문제가 해결되는 경우가 많습니다.
