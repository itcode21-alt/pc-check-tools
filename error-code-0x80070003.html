에러 코드 상세

0x80070003 · 지정된 경로를 찾을 수 없음

설치나 업데이트, 파일 접근 시 시스템이 지정된 경로 자체를 찾지 못했을 때 나타나는 오류로, 경로 손상이나 이동된 폴더가 주요 원인입니다.

이 코드를 어떻게 해석해야 하나요?

업데이트 관련 코드는 네트워크, 캐시, 구성 요소 저장소, 디스크 공간, 서비스 상태 중 어느 지점에서 작업이 중단됐는지를 보여줍니다. 실패가 반복되는 시점과 이벤트 로그의 세부 코드를 함께 확인하면 원인을 좁히기 쉽습니다.

먼저 기록할 내용: 0x80070003가 발생한 시각, 직전에 실행한 작업, 최근 설치한 드라이버·업데이트, 재부팅 뒤 같은 코드가 반복되는지를 적어 두세요.

가능성 높은 원인

  • 설치 대상 폴더가 삭제되었거나 다른 위치로 이동된 경우
  • 레지스트리에 등록된 경로 정보가 최신 상태와 맞지 않는 경우
  • 네트워크 드라이브 연결이 끊겨 경로에 접근할 수 없는 경우
  • 경로 이름에 시스템이 처리하지 못하는 특수 문자가 포함된 경우
  • 이전 업데이트나 설치 과정에서 폴더 구조가 일부 손상된 경우

첫 점검 항목

  1. 오류 메시지에 표시된 경로가 실제로 존재하는지 탐색기에서 확인하세요.
  2. 네트워크 드라이브라면 연결이 살아있는지 확인하고 다시 매핑해 보세요.
  3. SoftwareDistribution 캐시를 초기화한 뒤 업데이트를 다시 시도하세요.
  4. sfc /scannow로 시스템 파일과 경로 관련 손상을 점검하세요.
  5. 가능하다면 문제 프로그램을 제거 후 재설치해 경로 정보를 새로 등록하세요.

데이터와 시스템을 보호하는 주의사항

시스템 폴더 권한 변경이나 레지스트리 정리 도구 사용보다 저장 공간, 네트워크, 서비스 상태를 먼저 확인하세요.

앞 단계로 해결되지 않을 때

같은 코드가 반복되면 Windows Update 기록과 DISM·SFC 결과를 저장하고 설치 미디어의 버전이 현재 Windows와 맞는지 확인하세요.

복구, 초기화, 드라이버 제거처럼 되돌리기 어려운 작업 전에는 중요한 파일을 다른 물리 저장장치나 신뢰할 수 있는 백업 위치에 보관하세요.

관련 증상 가이드 확인하기

같은 증상과 연결된 다른 오류 코드

  • 0x800F081F · 원본 파일을 찾을 수 없음
  • 0x80070422 · 필요한 서비스가 비활성화됨
  • 0x80070070 · 디스크 공간 부족
  • 0xC1900101 · 업데이트 드라이버 호환성 실패

자주 묻는 질문

  • 업데이트를 계속 재시도해도 괜찮은가요? 이 문제(0x80070003)로 반복 실패한다면 무한정 재시도하기보다 업데이트 구성 요소 정리와 디스크 공간 확인을 먼저 하는 것이 시간을 아낄 수 있습니다.
  • 수동으로 업데이트를 받아 설치해도 되나요? Microsoft Update 카탈로그에서 해당 KB 번호를 직접 받아 설치하면 자동 업데이트 오류를 우회할 수 있는 경우가 있습니다.
