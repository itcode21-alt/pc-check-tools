오류 1618 · 다른 설치가 진행 중입니다 (오류 1618)

이미 다른 Windows Installer 작업이 진행 중이라 새 설치나 제거를 시작할 수 없을 때 나타나는 오류입니다. 실제로는 이전 설치가 멈춰 있거나 비정상 종료된 잔여 프로세스인 경우가 많습니다.

이 코드를 어떻게 해석해야 하나요?

업데이트 관련 코드는 네트워크, 캐시, 구성 요소 저장소, 디스크 공간, 서비스 상태 중 어느 지점에서 작업이 중단됐는지를 보여줍니다. 실패가 반복되는 시점과 이벤트 로그의 세부 코드를 함께 확인하면 원인을 좁히기 쉽습니다.

먼저 기록할 내용: 오류 1618가 발생한 시각, 직전에 실행한 작업, 최근 설치한 드라이버·업데이트, 재부팅 뒤 같은 코드가 반복되는지를 적어 두세요.

가능성 높은 원인

  • 이전 설치/제거 작업이 완전히 끝나지 않고 백그라운드에 남아있는 경우
  • 여러 설치 프로그램을 동시에 실행한 경우
  • 이전 설치가 오류로 중단되며 Installer 프로세스가 종료되지 않은 경우
  • 자동 업데이트 프로그램이 백그라운드에서 Installer를 점유하고 있는 경우
  • 시스템 재부팅이 필요한 이전 작업이 완료되지 않은 경우

첫 점검 항목

  1. 작업 관리자에서 msiexec.exe 프로세스가 실행 중인지 확인하고, 있다면 종료 후 재시도하세요.
  2. PC를 재부팅한 뒤 다른 프로그램 없이 설치를 다시 시도하세요.
  3. 백그라운드 업데이트 프로그램(스토어, 자동 업데이트 유틸리티 등)을 일시 종료하세요.
  4. 동시에 여러 설치 프로그램을 실행하지 말고 하나씩 순서대로 진행하세요.
  5. 반복되면 시스템을 재부팅해 대기 중인 설치 작업을 완료시킨 뒤 다시 시도하세요.

데이터와 시스템을 보호하는 주의사항

시스템 폴더 권한 변경이나 레지스트리 정리 도구 사용보다 저장 공간, 네트워크, 서비스 상태를 먼저 확인하세요.

앞 단계로 해결되지 않을 때

같은 코드가 반복되면 Windows Update 기록과 DISM·SFC 결과를 저장하고 설치 미디어의 버전이 현재 Windows와 맞는지 확인하세요.

복구, 초기화, 드라이버 제거처럼 되돌리기 어려운 작업 전에는 중요한 파일을 다른 물리 저장장치나 신뢰할 수 있는 백업 위치에 보관하세요.

관련 증상 가이드 확인하기

같은 증상과 연결된 다른 오류 코드

  • 오류 2503 · 이 설치 패키지를 열 수 없습니다 (오류 2503)
  • 오류 2502 · 이 설치 패키지를 열 수 없습니다 (오류 2502)
  • 오류 0x80070643 · 설치 중 오류가 발생했습니다 (0x80070643)

자주 묻는 질문

  • 설치 오류가 나면 프로그램을 다시 받으면 되나요? 이 문제(오류 1618)는 설치 파일보다 Windows Installer 서비스나 잔여 파일이 원인인 경우가 많아, 재다운로드 전에 서비스 상태와 이전 설치 흔적부터 정리해보세요.
  • 이전 버전을 지우지 않고 업데이트 설치하면 안 되나요? 일부 프로그램은 가능하지만, 오류가 반복된다면 이전 버전을 완전히 제거한 뒤 재설치하는 것이 안전합니다.
