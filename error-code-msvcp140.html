MSVCP140.dll 오류 · MSVCP140.dll이 없어서 프로그램을 시작할 수 없습니다

Visual C++ 재배포 패키지에 포함된 런타임 라이브러리 파일이 없거나 손상되었을 때 나타나는 오류로, 게임이나 여러 프로그램에서 공통으로 요구하는 파일이라 매우 자주 발생합니다.

이 코드를 어떻게 해석해야 하나요?

업데이트 관련 코드는 네트워크, 캐시, 구성 요소 저장소, 디스크 공간, 서비스 상태 중 어느 지점에서 작업이 중단됐는지를 보여줍니다. 실패가 반복되는 시점과 이벤트 로그의 세부 코드를 함께 확인하면 원인을 좁히기 쉽습니다.

먼저 기록할 내용: MSVCP140.dll 오류가 발생한 시각, 직전에 실행한 작업, 최근 설치한 드라이버·업데이트, 재부팅 뒤 같은 코드가 반복되는지를 적어 두세요.

가능성 높은 원인

  • Visual C++ 재배포 패키지가 설치되어 있지 않은 경우
  • 설치된 재배포 패키지가 손상되었거나 버전이 맞지 않는 경우
  • 다른 프로그램을 제거하는 과정에서 공유 DLL 파일이 함께 삭제된 경우
  • 인터넷에서 개별적으로 받은 DLL 파일을 잘못된 위치에 넣은 경우
  • 시스템 파일 손상

첫 점검 항목

  1. 마이크로소프트 공식 홈페이지에서 Visual C++ 재배포 패키지(x86, x64 모두)를 새로 받아 설치하세요.
  2. 기존에 설치된 재배포 패키지가 있다면 먼저 제거한 뒤 깨끗하게 재설치하세요.
  3. DLL 파일을 인터넷에서 개별적으로 받아 넣는 방법은 권장하지 않습니다 — 출처가 불분명한 파일은 보안 위험이 있습니다.
  4. sfc /scannow로 시스템 파일 손상 여부를 점검하세요.
  5. 문제가 되는 프로그램을 재설치해 필요한 구성 요소를 함께 설치하세요.

데이터와 시스템을 보호하는 주의사항

시스템 폴더 권한 변경이나 레지스트리 정리 도구 사용보다 저장 공간, 네트워크, 서비스 상태를 먼저 확인하세요.

앞 단계로 해결되지 않을 때

같은 코드가 반복되면 Windows Update 기록과 DISM·SFC 결과를 저장하고 설치 미디어의 버전이 현재 Windows와 맞는지 확인하세요.

복구, 초기화, 드라이버 제거처럼 되돌리기 어려운 작업 전에는 중요한 파일을 다른 물리 저장장치나 신뢰할 수 있는 백업 위치에 보관하세요.

자주 묻는 질문

  • 앱이 실행되지 않으면 무조건 재설치해야 하나요? 이 문제(MSVCP140.dll 오류)는 재설치보다 런타임 구성 요소(비주얼 C++ 등) 문제인 경우가 많아, 재설치 전에 필수 구성 요소부터 점검하는 것이 좋습니다.
