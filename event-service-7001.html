EVENT ID 7001

Service Control Manager 7001 의존 서비스 실패 점검

의존하는 다른 서비스가 시작되지 않아 현재 서비스도 시작할 수 없음을 나타냅니다.

7001은 의존하는 다른 서비스가 시작되지 않아 현재 서비스도 함께 시작하지 못했다는 기록입니다. 이벤트 설명에 표시된 의존 서비스 이름을 먼저 확인해야 합니다.

실패한 의존 서비스를 서비스 관리자에서 수동으로 먼저 시작해보면, 그 서비스만의 문제인지 연쇄적인 문제인지 확인할 수 있습니다.

발생 조건

  • 부팅 또는 앱 시작 시

가능성 높은 원인

  • 선행 서비스 실패
  • 서비스 구성 손상

점검 순서

  1. 설명에 표시된 의존 서비스 확인 — 되돌리기 쉬운 항목부터 확인해 위험 없이 범위를 줄입니다.
  2. 7000·7023 이벤트와 시간 비교 — 첫 확인 결과와 비교해 원인 후보를 좁힙니다.
  3. 관련 앱 복구 — 앞 단계에서 해결되지 않을 때 다음 원인을 검증합니다.
주의사항

의존성 레지스트리를 임의 수정하지 마세요.

관련 코드·가이드

0x80070422 오류 코드 확인하기앱 실행 불가 가이드

자주 묻는 질문

  • 의존 서비스가 무엇인지 어떻게 확인하나요? 이벤트 설명에 실패한 의존 서비스 이름이 함께 표시됩니다. 서비스 관리자(services.msc)에서 해당 서비스의 상태와 시작 유형을 확인하세요.
  • 7000과 7001이 같이 뜨면 어느 것이 원인인가요? 먼저 발생한 7000의 서비스가 실패의 시작점일 가능성이 큽니다. 두 이벤트의 시각과 서비스 이름을 비교해 순서를 확인하세요.
