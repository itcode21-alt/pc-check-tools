storahci 이벤트 129

홈진단가이드문의

EVENT ID 129

장치 재설정이 반복되는 원인을 연결·펌웨어·전원 관리로 나누세요

저장장치 요청이 제한 시간 안에 끝나지 않아 Windows가 컨트롤러를 재설정했을 때 기록됩니다.

증상은 몇 초 동안 마우스까지 멈추거나, 게임 로딩이 길어지거나, 부팅 때 SSD 인식이 늦는 형태로 나타날 수 있습니다. 이벤트 원본은 storahci 외에도 stornvme, iaStor 계열처럼 컨트롤러와 드라이버에 따라 달라질 수 있으므로 ID와 원본을 함께 기록해야 합니다.

점검 순서

  1. 중요 데이터를 백업하고 같은 시각에 Disk 7·51·153이 있는지 확인합니다.
  2. SSD 제조사 도구로 펌웨어, 건강 상태, 온도와 오류 기록을 확인합니다.
  3. SATA 장치는 케이블과 포트를, NVMe는 슬롯과 방열 상태를 교차합니다.
  4. 메인보드 칩셋·스토리지 드라이버와 BIOS를 제조사 지원 기준으로 확인합니다.
  5. 절전 복귀에서만 발생하면 링크 전원 관리와 빠른 시작 영향을 비교합니다.

레지스트리 수정 전에 볼 것

인터넷에는 AHCI 링크 전원 관리나 시간 제한 값을 바꾸는 방법이 많지만, 값 변경은 증상을 숨기고 실제 장치 지연을 놓치게 할 수 있습니다. 먼저 장치 상태와 펌웨어, 연결을 확인하고 변경 전 값을 기록해야 합니다.

단발성과 반복의 차이

업데이트 직후 한 번 기록되고 실제 지연이 없다면 관찰할 수 있습니다. 같은 디스크에서 129와 153이 매일 늘거나 파일 복사와 부팅이 멈춘다면 백업 후 장치 교차 테스트가 필요합니다. 시스템 드라이브라면 오류가 커지기 전에 교체 계획을 세우는 편이 안전합니다.
