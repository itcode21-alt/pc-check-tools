에러 코드 상세

0x800705b4 · 작업 시간 초과

업데이트나 특정 작업이 정해진 시간 안에 완료되지 못하고 시간 초과로 중단됐을 때 나타나는 오류입니다. 느린 네트워크나 시스템 응답 지연이 주요 원인입니다.

이 코드를 어떻게 해석해야 하나요?

업데이트 관련 코드는 네트워크, 캐시, 구성 요소 저장소, 디스크 공간, 서비스 상태 중 어느 지점에서 작업이 중단됐는지를 보여줍니다. 실패가 반복되는 시점과 이벤트 로그의 세부 코드를 함께 확인하면 원인을 좁히기 쉽습니다.

먼저 기록할 내용: 0x800705b4가 발생한 시각, 직전에 실행한 작업, 최근 설치한 드라이버·업데이트, 재부팅 뒤 같은 코드가 반복되는지를 적어 두세요.

가능성 높은 원인

  • 네트워크 속도가 느리거나 불안정해 다운로드가 지연되는 경우
  • 디스크 응답 속도가 느려 파일 처리가 지연되는 경우
  • 백그라운드에서 다른 무거운 작업이 동시에 실행되고 있는 경우
  • 서버 측 응답이 느린 시간대에 접속한 경우
  • 보안 소프트웨어의 검사로 인해 처리 속도가 느려진 경우

첫 점검 항목

  1. 네트워크 상태를 확인하고 가능하면 더 안정적인 연결(유선 등)로 다시 시도하세요.
  2. 다른 무거운 프로그램을 종료한 뒤 작업을 다시 시도하세요.
  3. 디스크 상태(특히 HDD의 경우 조각모음, SSD의 경우 여유 공간)를 점검하세요.
  4. 시간대를 바꿔(예: 트래픽이 적은 새벽 시간) 다시 시도해 보세요.
  5. 보안 프로그램의 실시간 검사를 일시적으로 낮춘 뒤 재시도해 보세요.

데이터와 시스템을 보호하는 주의사항

시스템 폴더 권한 변경이나 레지스트리 정리 도구 사용보다 저장 공간, 네트워크, 서비스 상태를 먼저 확인하세요.

앞 단계로 해결되지 않을 때

같은 코드가 반복되면 Windows Update 기록과 DISM·SFC 결과를 저장하고 설치 미디어의 버전이 현재 Windows와 맞는지 확인하세요.

복구, 초기화, 드라이버 제거처럼 되돌리기 어려운 작업 전에는 중요한 파일을 다른 물리 저장장치나 신뢰할 수 있는 백업 위치에 보관하세요.

관련 증상 가이드 확인하기

같은 증상과 연결된 다른 오류 코드

  • 0x800F081F · 원본 파일을 찾을 수 없음
  • 0x80070422 · 필요한 서비스가 비활성화됨
  • 0x80070070 · 디스크 공간 부족
  • 0xC1900101 · 업데이트 드라이버 호환성 실패

공식 참고자료

Microsoft Learn: System Error Codes (1300-1699) — ERROR_TIMEOUT

자주 묻는 질문

  • 업데이트를 계속 재시도해도 괜찮은가요? 이 문제(0x800705b4)로 반복 실패한다면 무한정 재시도하기보다 업데이트 구성 요소 정리와 디스크 공간 확인을 먼저 하는 것이 시간을 아낄 수 있습니다.
  • 업데이트 오류 코드마다 해결법이 다른가요? 공통 원인(구성 요소 정리, 디스크 공간)이 많지만 코드별로 원인이 갈리는 경우도 있어, 코드 자체를 함께 검색해보는 것이 좋습니다.
