에러 코드 상세

0x80070570 · 파일이 손상되었거나 읽을 수 없음

설치나 복사 과정에서 파일 손상이나 읽기 실패가 생길 때 보일 수 있습니다.

이 코드를 어떻게 해석해야 하나요?

업데이트·설치 코드는 파일 누락, 구성 요소 저장소, 서비스, 권한, 여유 공간 중 작업이 중단된 지점을 나타냅니다. 실패 비율과 업데이트 기록의 세부 코드를 함께 봐야 합니다.

먼저 기록할 내용: 0x80070570가 발생한 시각, 직전에 실행한 작업, 최근 설치한 드라이버·업데이트, 재부팅 뒤 같은 코드가 반복되는지를 적어 두세요.

가능성 높은 원인

  • 설치 파일 손상
  • 디스크 불량
  • 압축 해제 오류

첫 점검 항목

  1. 파일을 다시 받기 — 되돌리기 쉬운 항목부터 확인해 위험 없이 범위를 줄입니다.
  2. 디스크 검사 — 첫 확인 결과와 비교해 소프트웨어와 하드웨어 가능성을 나눕니다.
  3. 다른 저장소에 저장 — 앞 단계에서 해결되지 않을 때 다음 원인 후보를 검증합니다.

데이터와 시스템을 보호하는 주의사항

시스템 폴더 권한 변경이나 레지스트리 정리 도구 사용보다 저장 공간, 네트워크, 서비스 상태를 먼저 확인하세요.

앞 단계로 해결되지 않을 때

같은 코드가 반복되면 Windows Update 기록과 DISM·SFC 결과를 저장하고 설치 미디어의 버전이 현재 Windows와 맞는지 확인하세요.

복구, 초기화, 드라이버 제거처럼 되돌리기 어려운 작업 전에는 중요한 파일을 다른 물리 저장장치나 신뢰할 수 있는 백업 위치에 보관하세요.

관련 증상 가이드 확인하기

같은 증상과 연결된 다른 오류 코드

  • 0x0000007B · INACCESSIBLE_BOOT_DEVICE
  • 0x00000133 · DPC_WATCHDOG_VIOLATION
  • 0x00000024 · NTFS_FILE_SYSTEM
  • 0x00000077 · KERNEL_STACK_INPAGE_ERROR

자주 묻는 질문

  • SSD·NVMe를 바로 교체해야 하나요? 이 문제(0x80070570)만으로 저장장치 고장을 단정하기는 어렵습니다. 슬롯 재장착과 펌웨어 업데이트를 먼저 확인한 뒤에도 반복되면 다른 슬롯이나 드라이브로 교차 테스트해보는 것이 안전합니다.
  • SSD 상태를 확인하는 무료 도구가 있나요? CrystalDiskInfo 같은 무료 프로그램으로 SSD의 건강 상태와 온도를 확인할 수 있습니다.
