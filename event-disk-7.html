Disk 이벤트 7

홈진단가이드게임문의

EVENT ID 7

검사보다 중요한 파일 백업을 먼저 진행하세요

Disk 7은 저장장치에서 읽을 수 없는 블록이 감지되었다는 기록으로, 반복된다면 데이터 손실 위험을 먼저 고려해야 합니다.

이벤트 메시지에는 Harddisk 번호가 표시될 수 있지만 탐색기의 C: 드라이브 문자와 바로 일치하지 않을 수 있습니다. 디스크 관리나 PowerShell에서 실제 물리 디스크 번호와 모델을 확인해야 잘못된 장치를 점검하는 일을 피할 수 있습니다.

안전한 순서

  1. 현재 열리는 중요한 파일부터 다른 물리 저장장치나 클라우드에 복사합니다.
  2. CrystalDiskInfo와 제조사 진단 도구에서 건강 상태, 재할당·보류·복구 불가 항목을 확인합니다.
  3. SATA 장치라면 데이터 케이블과 전원 커넥터를 교체하고 다른 포트에서 비교합니다.
  4. NVMe라면 M.2 장착, 방열판 압력, 온도, 펌웨어를 확인합니다.
  5. 백업을 끝낸 뒤 파일 시스템 검사와 표면 진단을 계획합니다.

이벤트 7이 한 번만 보일 때

외장 디스크를 갑자기 분리했거나 케이블 접촉이 잠시 불안정했을 수도 있습니다. 단발성이라면 같은 디스크에서 51, 55, 129, 153이 이어지는지와 실제 파일 읽기 오류가 있는지 관찰합니다. 반대로 같은 블록 오류가 늘거나 파일 복사가 멈춘다면 사용을 줄이는 편이 안전합니다.

하면 안 되는 실수

백업 전에 chkdsk 복구 옵션이나 쓰기 부하가 큰 검사를 반복하지 마세요. 파일 시스템을 정리하는 작업은 물리적으로 불안정한 장치에 추가 부담을 줄 수 있습니다. SMART가 정상이어도 이벤트와 실제 읽기 실패가 반복되면 장치가 정상이라고 단정할 수 없습니다.

교체를 검토할 신호

재할당·보류 섹터 증가, 제조사 진단 실패, 다른 케이블·포트에서도 같은 오류, 읽을 수 없는 파일이 함께 나타나면 교체 우선순위가 높습니다.

자주 묻는 질문

  • 이벤트 7이 한 번만 떴는데 디스크를 바로 교체해야 하나요? 단발성이라면 즉시 교체보다 관찰이 우선입니다. 외장 디스크를 갑자기 분리했거나 케이블 접촉이 잠시 불안정했을 가능성이 있습니다. 다만 같은 디스크에서 51, 55, 129, 153이 함께 늘거나 실제 파일 읽기 오류가 있다면 교체를 앞당기세요.
  • SMART가 정상인데 이벤트 7이 계속 뜨는 이유는 무엇인가요? SMART는 제조사가 정의한 임계값을 넘었을 때만 경고하므로, 임계값 이전 단계의 배드 섹터나 케이블·연결 문제는 정상으로 표시될 수 있습니다. SMART가 정상이어도 이벤트와 실제 읽기 실패가 반복되면 장치가 완전히 정상이라고 단정할 수 없습니다.
  • 외장 하드에서도 이벤트 7이 발생하나요? 네, USB로 연결된 외장 저장장치도 같은 이벤트를 남길 수 있습니다. 이 경우 디스크 자체 문제 외에 USB 케이블, 허브, 포트 전원 공급 불안정도 흔한 원인이므로 함께 확인하세요.
