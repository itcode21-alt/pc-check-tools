    복사·붙여넣기 안됨 원인과 점검 순서

    Ctrl+C/V가 동작하지 않거나, 클립보드 내용이 붙여넣어지지 않는 경우

    클립보드 오류는 rdpclip.exe 프로세스 충돌, 특정 앱의 클립보드 잠금, 또는 클립보드를 감시하는 보안·유틸리티 프로그램과의 충돌이 주요 원인입니다. 재부팅 없이 해당 프로세스만 재시작해 해결되는 경우가 많습니다.

    원격 데스크톱(RDP) 환경에서는 클립보드 동기화 설정이 따로 있어 이 설정이 꺼져 있으면 복사·붙여넣기가 동작하지 않습니다.

    이 가이드가 필요한 상황

    • Ctrl+C로 복사한 내용이 Ctrl+V로 붙여넣어지지 않는다.
    • 클립보드 내용이 잠시 후 자동으로 지워진다.
    • 특정 앱에서만 붙여넣기가 안 된다.
    • Win+V를 눌러도 클립보드 기록이 비어 있다.

    먼저 확인할 항목

    1. rdpclip.exe 재시작 — 작업 관리자(Ctrl+Shift+Esc)에서 rdpclip.exe를 찾아 종료합니다. 이후 파일 → 새 작업 실행에서 rdpclip을 입력해 다시 시작합니다.
    2. 다른 앱에서도 재현되는지 확인 — 특정 앱에서만 안 된다면 해당 앱 자체의 문제입니다. 메모장에서 복사·붙여넣기가 정상이면 앱 설정이나 권한을 먼저 점검하세요.
    3. 클립보드 기록 활성화 확인 — 설정 → 시스템 → 클립보드에서 클립보드 기록이 켜져 있는지 확인하고, Win+V로 기록이 쌓이는지 테스트합니다.
    4. 최근 설치한 프로그램 점검 — 클립보드를 모니터링하는 유틸리티(비밀번호 관리자, 번역 프로그램, 화면 캡처 툴)가 충돌을 일으키는 경우가 있습니다. 최근 설치 목록을 확인하세요.

    원인을 더 정확히 나누는 방법

    모든 앱에서 안 되는 경우 vs 특정 앱만

    메모장에서도 복사·붙여넣기가 안 된다면 클립보드 서비스 자체의 문제입니다. 특정 앱에서만 안 된다면 해당 앱의 권한이나 클립보드 접근 차단 설정을 먼저 봐야 합니다.

    원격 데스크톱 환경의 경우

    RDP로 연결된 PC에서 클립보드가 안 된다면, 원격 데스크톱 연결 설정에서 로컬 리소스 탭의 클립보드 항목이 체크되어 있는지 확인하세요. 서버 측 그룹 정책에서 클립보드를 비활성화한 경우에는 관리자에게 문의해야 합니다.

    확인 결과에 따른 다음 단계

    • rdpclip 재시작 후 해결될 때 — 일시적 충돌이었으므로 반복 시 재부팅으로 해결합니다.
    • 특정 프로그램 삭제 후 해결될 때 — 해당 프로그램이 클립보드를 독점하고 있었으므로 대체 프로그램을 찾거나 설정을 조정하세요.
    • sfc /scannow 실행 — 위 방법으로 해결되지 않으면 관리자 권한 명령 프롬프트에서 sfc /scannow를 실행해 시스템 파일 손상을 점검합니다.

    자주 묻는 질문

    • 복사·붙여넣기가 갑자기 안 되면 어떻게 하나요? 작업 관리자에서 rdpclip.exe 프로세스를 찾아 종료한 뒤 새 작업 실행에서 rdpclip을 다시 실행해 보세요.
    • 클립보드 내용이 계속 지워지면? 백신이나 클립보드 모니터링 프로그램이 원인일 수 있습니다. Win+V로 클립보드 기록을 확인하고 최근 설치한 프로그램을 점검하세요.
