에러 코드 상세

이 앱이 PC에서 실행되지 않습니다 · 이 앱이 사용자의 PC에서 실행되지 않습니다

프로그램의 아키텍처(32비트/64비트)가 현재 시스템과 맞지 않거나, 윈도우 버전 호환성 문제로 실행이 차단될 때 나타나는 메시지입니다.

이 코드를 어떻게 해석해야 하나요?

업데이트 관련 코드는 네트워크, 캐시, 구성 요소 저장소, 디스크 공간, 서비스 상태 중 어느 지점에서 작업이 중단됐는지를 보여줍니다. 실패가 반복되는 시점과 이벤트 로그의 세부 코드를 함께 확인하면 원인을 좁히기 쉽습니다.

먼저 기록할 내용: 이 앱이 PC에서 실행되지 않습니다가 발생한 시각, 직전에 실행한 작업, 최근 설치한 드라이버·업데이트, 재부팅 뒤 같은 코드가 반복되는지를 적어 두세요.

가능성 높은 원인

  • 32비트 전용 프로그램을 지원하지 않는 환경에서 실행하려는 경우 (드물게)
  • 프로그램이 요구하는 윈도우 버전보다 낮거나 호환되지 않는 버전을 사용 중인 경우
  • 다운로드한 설치 파일이 손상되었거나 위조된 경우
  • 스마트스크린이나 보안 정책이 실행을 차단한 경우
  • 그룹 정책으로 특정 프로그램 실행이 제한된 경우

첫 점검 항목

  1. 프로그램 공식 홈페이지에서 시스템 요구 사항(윈도우 버전, 32/64비트)을 확인하세요.
  2. 설치 파일을 다시 받아 무결성을 확인한 뒤 재설치하세요.
  3. 메시지가 뜬 파일을 우클릭 > 속성에서 '차단 해제' 옵션이 있다면 체크하세요.
  4. 윈도우를 최신 버전으로 업데이트한 뒤 다시 시도하세요.
  5. 회사/학교 PC라면 그룹 정책으로 제한된 프로그램인지 확인하세요.

데이터와 시스템을 보호하는 주의사항

시스템 폴더 권한 변경이나 레지스트리 정리 도구 사용보다 저장 공간, 네트워크, 서비스 상태를 먼저 확인하세요.

앞 단계로 해결되지 않을 때

같은 코드가 반복되면 Windows Update 기록과 DISM·SFC 결과를 저장하고 설치 미디어의 버전이 현재 Windows와 맞는지 확인하세요.

복구, 초기화, 드라이버 제거처럼 되돌리기 어려운 작업 전에는 중요한 파일을 다른 물리 저장장치나 신뢰할 수 있는 백업 위치에 보관하세요.

관련 증상 가이드 확인하기

같은 증상과 연결된 다른 오류 코드

  • 오류 1603 · 설치 중 치명적 오류가 발생했습니다 (오류 1603)
  • 오류 1618 · 다른 설치가 진행 중입니다 (오류 1618)
  • 오류 2503 · 이 설치 패키지를 열 수 없습니다 (오류 2503)
  • 오류 2502 · 이 설치 패키지를 열 수 없습니다 (오류 2502)

자주 묻는 질문

  • 앱이 실행되지 않으면 무조건 재설치해야 하나요? 이 문제(이 앱이 PC에서 실행되지 않습니다)는 재설치보다 런타임 구성 요소(비주얼 C++ 등) 문제인 경우가 많아, 재설치 전에 필수 구성 요소부터 점검하는 것이 좋습니다.
  • 호환성 모드로 실행하면 도움이 되나요? 오래된 프로그램이라면 호환성 모드가 도움이 될 수 있지만, 최신 프로그램의 오류라면 다른 원인을 먼저 점검해야 합니다.
