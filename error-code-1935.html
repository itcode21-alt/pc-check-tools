오류 1935 · 어셈블리 구성 요소 설치 중 오류 (오류 1935)

설치 프로그램이 .NET이나 Visual C++ 재배포 패키지 같은 어셈블리 구성 요소를 설치하는 과정에서 실패했을 때 나타나는 오류입니다.

이 코드를 어떻게 해석해야 하나요?

업데이트 관련 코드는 네트워크, 캐시, 구성 요소 저장소, 디스크 공간, 서비스 상태 중 어느 지점에서 작업이 중단됐는지를 보여줍니다. 실패가 반복되는 시점과 이벤트 로그의 세부 코드를 함께 확인하면 원인을 좁히기 쉽습니다.

먼저 기록할 내용: 오류 1935가 발생한 시각, 직전에 실행한 작업, 최근 설치한 드라이버·업데이트, 재부팅 뒤 같은 코드가 반복되는지를 적어 두세요.

가능성 높은 원인

  • Visual C++ 재배포 패키지나 .NET Framework가 손상된 경우
  • Windows Installer 서비스가 손상된 경우
  • 시스템 파일 손상으로 구성 요소 등록에 실패하는 경우
  • 보안 소프트웨어가 구성 요소 설치를 차단하는 경우
  • 디스크 배드섹터로 파일 일부가 손상된 경우

첫 점검 항목

  1. 제어판에서 기존 Visual C++ 재배포 패키지를 제거한 뒤 최신 버전을 다시 설치하세요.
  2. .NET Framework를 복구하거나 재설치해 보세요.
  3. sfc /scannow와 DISM 복구 명령으로 시스템 파일을 점검하세요.
  4. 보안 프로그램을 임시로 비활성화한 뒤 재시도하세요.
  5. 디스크 상태를 chkdsk로 점검하세요.

데이터와 시스템을 보호하는 주의사항

시스템 폴더 권한 변경이나 레지스트리 정리 도구 사용보다 저장 공간, 네트워크, 서비스 상태를 먼저 확인하세요.

앞 단계로 해결되지 않을 때

같은 코드가 반복되면 Windows Update 기록과 DISM·SFC 결과를 저장하고 설치 미디어의 버전이 현재 Windows와 맞는지 확인하세요.

복구, 초기화, 드라이버 제거처럼 되돌리기 어려운 작업 전에는 중요한 파일을 다른 물리 저장장치나 신뢰할 수 있는 백업 위치에 보관하세요.

관련 증상 가이드 확인하기

같은 증상과 연결된 다른 오류 코드

  • 오류 1603 · 설치 중 치명적 오류가 발생했습니다 (오류 1603)
  • 오류 1618 · 다른 설치가 진행 중입니다 (오류 1618)
  • 오류 2503 · 이 설치 패키지를 열 수 없습니다 (오류 2503)
  • 오류 2502 · 이 설치 패키지를 열 수 없습니다 (오류 2502)

자주 묻는 질문

  • 설치 오류가 나면 프로그램을 다시 받으면 되나요? 이 문제(오류 1935)는 설치 파일보다 Windows Installer 서비스나 잔여 파일이 원인인 경우가 많아, 재다운로드 전에 서비스 상태와 이전 설치 흔적부터 정리해보세요.
  • 설치 오류가 백신 때문인지 어떻게 확인하나요? 백신을 일시적으로 끄고 설치를 시도해 보면 원인이 백신인지 아닌지 빠르게 확인할 수 있습니다.
