에러 코드 상세

0x80070718 · IP 주소를 가져올 수 없음

DHCP 서버로부터 IP 주소를 할당받지 못했을 때 나타나는 오류로, 공유기의 DHCP 기능 문제나 네트워크 혼잡이 흔한 원인입니다.

이 코드를 어떻게 해석해야 하나요?

업데이트 관련 코드는 네트워크, 캐시, 구성 요소 저장소, 디스크 공간, 서비스 상태 중 어느 지점에서 작업이 중단됐는지를 보여줍니다. 실패가 반복되는 시점과 이벤트 로그의 세부 코드를 함께 확인하면 원인을 좁히기 쉽습니다.

먼저 기록할 내용: 0x80070718가 발생한 시각, 직전에 실행한 작업, 최근 설치한 드라이버·업데이트, 재부팅 뒤 같은 코드가 반복되는지를 적어 두세요.

가능성 높은 원인

  • 공유기의 DHCP 서버 기능이 꺼져 있거나 오류가 발생한 경우
  • 네트워크에 연결된 기기가 많아 할당 가능한 IP 주소가 소진된 경우
  • 네트워크 어댑터의 IP 설정이 잘못 구성된 경우
  • 이전에 할당받은 IP 주소가 충돌을 일으키는 경우
  • 네트워크 드라이버 문제로 DHCP 요청 자체가 실패하는 경우

첫 점검 항목

  1. 공유기를 재시작해 DHCP 서버 기능을 초기화하세요.
  2. 명령 프롬프트(관리자)에서 ipconfig /release 후 ipconfig /renew를 실행해 IP를 다시 요청하세요.
  3. 네트워크 어댑터 속성에서 IP 주소를 자동으로 받도록 설정되어 있는지 확인하세요.
  4. 공유기 관리 화면에서 DHCP 주소 범위와 현재 연결된 기기 수를 확인하세요.
  5. 네트워크 드라이버를 최신 버전으로 업데이트한 뒤 다시 시도하세요.

데이터와 시스템을 보호하는 주의사항

시스템 폴더 권한 변경이나 레지스트리 정리 도구 사용보다 저장 공간, 네트워크, 서비스 상태를 먼저 확인하세요.

앞 단계로 해결되지 않을 때

같은 코드가 반복되면 Windows Update 기록과 DISM·SFC 결과를 저장하고 설치 미디어의 버전이 현재 Windows와 맞는지 확인하세요.

복구, 초기화, 드라이버 제거처럼 되돌리기 어려운 작업 전에는 중요한 파일을 다른 물리 저장장치나 신뢰할 수 있는 백업 위치에 보관하세요.

관련 증상 가이드 확인하기

같은 증상과 연결된 다른 오류 코드

  • 0x800704CF · 이 네트워크에 연결되어 있지 않습니다
  • 0x80070490 · 요소를 찾을 수 없음 (네트워크 어댑터)
  • 0x80070102 · 네트워크 장치가 연결되지 않음
  • 배틀넷 연결 오류 · 배틀넷 오류로 오버워치 등이 실행되지 않음 (BLZBNTBNA0000)

자주 묻는 질문

  • 공유기를 바꾸면 해결되나요? 이 문제(0x80070718)는 공유기 자체 문제보다 어댑터 드라이버나 전원 관리 설정이 원인인 경우가 많아, 장비 교체 전에 드라이버 업데이트와 절전 설정부터 확인해보세요.
  • 5GHz와 2.4GHz 중 뭐가 더 안정적인가요? 5GHz는 속도가 빠르지만 벽 등 장애물에 약해 거리가 멀면 오히려 끊김이 잦을 수 있어, 상황에 따라 2.4GHz가 더 안정적일 수 있습니다.
