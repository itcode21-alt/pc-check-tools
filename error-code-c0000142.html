에러 코드 상세

0xc0000142 · 응용 프로그램을 정상적으로 시작하지 못했습니다 (0xc0000142)

프로그램이 실행되는 과정에서 필요한 초기화 작업을 완료하지 못했을 때 나타나는 매우 흔한 오류입니다. DLL 충돌이나 손상된 사용자 프로필, 그래픽 드라이버 문제가 대표적인 원인입니다.

이 코드를 어떻게 해석해야 하나요?

업데이트 관련 코드는 네트워크, 캐시, 구성 요소 저장소, 디스크 공간, 서비스 상태 중 어느 지점에서 작업이 중단됐는지를 보여줍니다. 실패가 반복되는 시점과 이벤트 로그의 세부 코드를 함께 확인하면 원인을 좁히기 쉽습니다.

먼저 기록할 내용: 0xc0000142가 발생한 시각, 직전에 실행한 작업, 최근 설치한 드라이버·업데이트, 재부팅 뒤 같은 코드가 반복되는지를 적어 두세요.

가능성 높은 원인

  • 프로그램이 의존하는 DLL 파일이 손상되었거나 버전이 맞지 않는 경우
  • 그래픽 드라이버가 오래되었거나 프로그램과 호환되지 않는 경우
  • 바이러스 백신이 프로그램의 정상적인 초기화 과정을 차단하는 경우
  • 사용자 프로필이나 임시 폴더에 문제가 있는 경우
  • 프로그램 설치 파일 자체가 손상된 경우

첫 점검 항목

  1. PC를 재부팅한 뒤 프로그램을 다시 실행해 보세요.
  2. 그래픽 드라이버를 제조사 최신 버전으로 업데이트하세요.
  3. 백신 프로그램을 임시로 비활성화한 뒤 재시도해 보세요.
  4. 프로그램을 관리자 권한으로 실행해 보세요.
  5. 문제가 계속되면 프로그램을 완전히 제거한 뒤 재설치해 보세요.

데이터와 시스템을 보호하는 주의사항

시스템 폴더 권한 변경이나 레지스트리 정리 도구 사용보다 저장 공간, 네트워크, 서비스 상태를 먼저 확인하세요.

앞 단계로 해결되지 않을 때

같은 코드가 반복되면 Windows Update 기록과 DISM·SFC 결과를 저장하고 설치 미디어의 버전이 현재 Windows와 맞는지 확인하세요.

복구, 초기화, 드라이버 제거처럼 되돌리기 어려운 작업 전에는 중요한 파일을 다른 물리 저장장치나 신뢰할 수 있는 백업 위치에 보관하세요.

관련 증상 가이드 확인하기

같은 증상과 연결된 다른 오류 코드

  • 오류 1603 · 설치 중 치명적 오류가 발생했습니다 (오류 1603)
  • 오류 1618 · 다른 설치가 진행 중입니다 (오류 1618)
  • 오류 2503 · 이 설치 패키지를 열 수 없습니다 (오류 2503)
  • 오류 2502 · 이 설치 패키지를 열 수 없습니다 (오류 2502)

공식 참고자료

Microsoft Learn: Application Failed to Initialize (0xc0000142)

자주 묻는 질문

  • 앱이 실행되지 않으면 무조건 재설치해야 하나요? 이 문제(0xc0000142)는 재설치보다 런타임 구성 요소(비주얼 C++ 등) 문제인 경우가 많아, 재설치 전에 필수 구성 요소부터 점검하는 것이 좋습니다.
  • 필수 구성 요소(Visual C++ 등)를 어떻게 확인하나요? 제어판의 프로그램 목록에서 Visual C++ Redistributable이 설치되어 있는지 확인하고, 없다면 Microsoft 공식 사이트에서 받아 설치해야 합니다.
