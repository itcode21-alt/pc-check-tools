코드 38 · 이 장치가 제거되었습니다

Windows가 장치를 인식했지만 현재 연결되어 있지 않거나, 물리적으로 분리된 상태를 나타냅니다. USB 외장 드라이브, 외장 하드디스크, 외부 메모리 장치에서 자주 보입니다.

이 코드를 어떻게 해석해야 하나요?

코드 38은 비교적 간단한 코드입니다. Windows가 과거에 장치를 찾은 적이 있지만, 지금은 물리적으로 연결되어 있지 않거나 전원이 공급되지 않고 있다는 의미입니다. USB, 외장 드라이브, 프린터, 스캐너 등에서 나타날 수 있습니다.

먼저 기록할 내용: 어느 장치에서 이 오류가 발생하는지, 장치가 실제로 연결되어 있는지, 전원이 공급되고 있는지를 확인하세요.

가능성 높은 원인

  • 장치가 물리적으로 분리되었음 (USB 케이블이 뽑혀있음, 외장 드라이브가 분리됨)
  • 장치 전원이 꺼져있음 (외장 하드디스크, 프린터 등)
  • USB 포트의 접촉 불량 (케이블이 느슨하게 연결됨)
  • USB 허브의 전원 미공급 (허브에 전원이 연결되지 않음)
  • 장치 드라이버만 남아있고 장치는 제거됨 (과거에 사용했던 장치)

첫 점검 항목 (우선순위 순서)

  1. 장치 물리적 연결 확인 — USB 케이블이 PC와 장치에 제대로 연결되어 있는지 확인합니다. 케이블이 헐거우면 더 깊게 끼웁니다.
  2. 장치 전원 확인 — 프린터, 외장 드라이브, 스캐너 등 독립적인 전원이 필요한 장치는 전원이 켜져 있는지 확인합니다.
  3. 다른 USB 포트 시도 — 다른 USB 포트(특히 뒤쪽 포트)에 연결해봅니다. 앞쪽 USB 포트가 느슨할 수 있습니다.
  4. 케이블 교체 — USB 케이블이 손상되었을 수 있으므로 다른 케이블을 시도합니다.
  5. USB 허브 확인 — USB 허브를 사용 중이면 허브 자체의 전원이 켜져 있는지, 허브가 PC에 제대로 연결되어 있는지 확인합니다.

USB 허브 사용 중 문제

외장 장치를 USB 허브에 연결하고 있다면, PC에 USB 허브를 직접 연결하는 것이 먼저 문제 해결의 핵심입니다. 허브가 충분한 전원을 공급하지 못하거나 허브 자체에 문제가 있을 수 있으므로, 허브를 우회해 PC에 직접 연결해보세요.

혼자 떨어진 드라이버

과거에 연결했던 USB 장치가 더 이상 없다면, 이 코드 38이 계속 나타나는 것이 정상입니다. 장치를 제거한 뒤 (오른쪽 클릭 → 장치 제거)하면 에러가 사라집니다. 이 경우 해결이 필요하지 않습니다.

데이터와 시스템을 보호하는 주의사항

외장 드라이브가 코드 38을 보이고 있다면, 연결되기 전까지 데이터를 건드리지 마세요. 드라이브의 파일에 접근하려면 먼저 제대로 연결하고 인식시켜야 합니다.

앞 단계로 해결되지 않을 때

모든 점검을 완료했는데도 장치가 인식되지 않으면:

  • 다른 PC에서 같은 장치와 케이블을 시도 (장치 자체 고장인지 PC 문제인지 구분)
  • 장치 제조사 기술 지원에 연락
  • PC의 BIOS에서 USB 포트가 활성화되었는지 확인

자주 묻는 질문

  • 코드 38이 나타나면 장치가 고장난 건가요? 아닙니다. 이 코드는 보통 물리적 연결 문제를 나타냅니다. 장치를 제대로 연결하고 전원을 확인한 후 다시 인식시키면 해결됩니다.
  • 이미 제거된 장치의 코드 38을 없애려면? 장치 관리자에서 해당 장치(노란 느낌표)를 마우스 오른쪽으로 클릭해 "장치 제거"를 선택하면 됩니다.
