하드웨어 오류

WHEA-Logger 18

홈진단가이드게임문의

EVENT ID 18

오류 유형과 발생 조건으로 CPU·메모리 범위를 나누세요

WHEA 18은 수정할 수 없는 하드웨어 오류를 Windows가 보고한 기록으로, 반복된다면 백업과 기본 설정 확인이 우선입니다.

이벤트 본문의 오류 유형이 Cache Hierarchy, Bus/Interconnect, Memory처럼 표시될 수 있고 Processor APIC ID가 함께 기록되기도 합니다. 그러나 APIC ID 하나만으로 CPU 불량을 확정할 수는 없습니다. 전압 설정, 메모리 컨트롤러, BIOS, 온도와 전원 공급도 같은 오류를 만들 수 있기 때문입니다.

먼저 해야 할 일

  1. 저장 중 재부팅이 반복되면 중요한 파일을 다른 저장장치에 백업합니다.
  2. BIOS에서 CPU 오버클럭, 언더볼팅, PBO, XMP·EXPO를 기본값으로 되돌립니다.
  3. 이벤트의 Error Type, APIC ID, 발생 시각을 기록하고 같은 작업에서 반복되는지 확인합니다.
  4. CPU와 GPU 온도, 메인보드 전원부 냉각, EPS 전원 연결을 확인합니다.
  5. 메모리를 한 개씩 기본 속도로 장착해 슬롯과 모듈을 교차합니다.

상황별 해석

게임이나 렌더링에서만 발생하면 발열과 전원 공급, 특정 코어 부하를 먼저 봅니다. 유휴 상태에서 발생한다면 절전 전환, 언더볼팅, BIOS 전원 관리도 확인해야 합니다. 메모리 테스트가 통과해도 XMP 상태에서만 오류가 난다면 모듈 불량이 아니라 메모리 컨트롤러와 설정 조합 문제일 수 있습니다.

피해야 할 방법

오류를 숨기기 위해 WHEA 서비스를 끄거나 전압을 임의로 높이지 마세요. 원인을 모른 채 장시간 스트레스 테스트를 반복하면 데이터 손상과 과열 위험이 커질 수 있습니다. 기본 설정에서도 반복될 때 제조사 진단과 부품별 교차 테스트를 진행하는 편이 안전합니다.

연결해서 볼 기록

Kernel-Power 41은 재부팅 결과, BugCheck 0x124는 하드웨어 오류 정지 코드입니다. 세 기록의 시각이 같다면 하나의 장애 흐름으로 비교하세요.

실제 사용자 사례

위 점검 순서로도 해결되지 않았을 때, 다른 원인으로 해결된 사례입니다.

CPU 교체로 해결한 사례

블루스크린 0x00000124와 함께 이 이벤트(WHEA-Logger 18)가 반복적으로 기록됐던 사례가 있습니다. 오버클럭 해제, 메모리 교차 테스트, 온도·전원 점검을 모두 진행했지만 해결되지 않다가 CPU를 교체한 뒤에야 문제가 해결됐습니다.

포인트: 이 이벤트가 반복 기록된다면, 드라이버나 설정보다 CPU 자체의 결함을 먼저 의심해보는 것이 좋습니다.

자주 묻는 질문

  • WHEA 18이 뜨면 CPU를 바로 교체해야 하나요? 아닙니다. Processor APIC ID가 기록되어도 CPU 불량을 바로 의미하지는 않습니다. 전압 설정, 메모리 컨트롤러, BIOS 설정, 온도, 전원 공급도 같은 오류를 만들 수 있으므로 오버클럭·언더볼팅을 기본값으로 되돌리고 온도와 전원부터 점검한 뒤에도 반복될 때 CPU 자체를 의심하세요.
  • 메모리 테스트를 통과했는데도 계속 발생하는 이유는 무엇인가요? 일반 메모리 테스트는 기본 속도에서 진행되는 경우가 많아 XMP·EXPO로 오버클럭된 상태의 불안정성을 잡아내지 못할 수 있습니다. XMP를 켠 상태에서만 오류가 난다면 모듈 불량이 아니라 메모리 컨트롤러와 오버클럭 설정 조합 문제일 가능성이 큽니다.
  • 오버클럭을 하지 않았는데도 WHEA 18이 발생할 수 있나요? 네, 가능합니다. 메인보드 제조사가 기본값으로 설정한 자동 전압·부스트 동작이 사실상 오버클럭처럼 작동하는 경우가 있고, 전원부 발열이나 BIOS 펌웨어 버그, 노후화된 부품 자체의 결함으로도 발생할 수 있습니다.
